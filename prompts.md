# Prompts utilizados (versión inicial)

Este documento recoge los prompts más relevantes utilizados durante la fase de definición del MVP.

---

## 📌 Producto

**Prompt:**
Define un MVP para un sistema de administración de taller con capacidades de OCR, cotización y generación de PDF, limitando el alcance a un flujo E2E funcional.

**Ajuste humano:**
Se redujo el alcance eliminando módulos como reportes, multi-sucursal e IA avanzada, priorizando un flujo completo funcional.

---

## 📌 Arquitectura

**Prompt:**
Diseña una arquitectura simple basada en Next.js con backend integrado, base de datos PostgreSQL y soporte para OCR.

**Ajuste humano:**
Se decidió utilizar Supabase para acelerar el desarrollo y evitar configuración de infraestructura.

---

## 📌 Modelo de datos

**Prompt:**
Define un modelo de datos mínimo para un sistema de check-in de vehículos y generación de cotizaciones.

---

## 📌 IA y herramientas

**Prompt:**
Cómo integrar herramientas de IA en el desarrollo de un MVP sin sobreingeniería.

**Ajuste humano:**
Se decidió utilizar herramientas como OpenCode y Mistral de forma asistida, validando manualmente los resultados.

---

Este documento será ampliado en siguientes iteraciones del proyecto conforme se incorporen más funcionalidades y uso de IA.
